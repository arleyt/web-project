Controle de Versão

Sistemas de controle de versão, são ferramentas utilizadas para gerenciar o versionamento de código. Seu principal objetivo e manter um histórico de todas as alterações realizadas durante o desenvolvimento de um software, por exemplo. Contudo, vale ressaltar que o versionamento pode ser realizado por arquivos, imagens, etc. Para o versionamento de código, abordarei sobre o Git e o Github.

Git

O Git é um software open-source, ou seja, de código aberto que visa gerenciar o controle de versão de projetos de software.

Github

Por sua vez, o Github é um software que utiliza o Git para controle de versão de código, altamente popular.

Testes

Testes no contexto de desenvolvimento de software, geralmente são utilizados para testar partes de um software com o objetivo de cobrir a maior partes dos casos de teste possíveis. Como resultado, o software desenvolvido passa a ter mais qualidade, aumentando a manutenibiliadde e refatoração de código. Diante disso, existem vários tipos de testes de software, contudo, vou abordar apenas a técnica test-driven development (TDD).

TDD

O TDD é o desenvolvimento orientado por testes, o qual visa em primeiro desenvolver um teste para um nova funcionalidade que vai ser desenvolvida. (ver artigo aqui).

Pytest

O Pytest é uma ferramenta que permite a realização de testes de software para a linguagem de programação python. Para mais detalhes sobre o framework, acesse a sua documentação aqui

Flask Framework

O Flask framework, é uma ferramenta em python com o objetivo de proporcionar a criação de aplicações web pequenas. Uma de suas vantagens é:

  • Simplicidade;
  • Rapidez;
  • Pequenas soluções;
